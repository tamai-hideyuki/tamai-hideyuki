# 型注釈と型推論

## 概要
- TypeScriptは値から型を推論できる
- 迷うところだけ明示的に型注釈を付けると読みやすい

## 最小例
```ts
let count = 0; // 推論: number
const title = "Hello"; // 推論: "Hello"（リテラル）

let score: number = 10; // 明示的な型注釈
```

## 使いどころ
- APIレスポンスや外部入力は型注釈で明示する
- 内部計算は推論に任せると簡潔

## 落とし穴
- `let` は再代入できるためリテラル型になりにくい
- `const` は再代入できないためリテラル型になりやすい

## 補足: const がリテラル型になりやすい理由
- `const` は再代入不可なので「この値のまま」と判断できる
- その結果、値そのもの（例: `"dark"`）が型になる

```ts
const mode = "dark"; // 型: "dark"
let theme = "dark";  // 型: string
```
