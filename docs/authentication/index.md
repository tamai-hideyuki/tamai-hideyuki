# 主要なプロバイダのOAuth認証とユーザー情報取得仕様まとめ

## 概要

このリポジトリは、主要なオンラインサービス（Google, Apple, Yahoo! JAPAN, Alipay）におけるOAuth認証フローと、認証後にユーザー情報を取得するためのAPIエンドポイントおよびその仕様について、まとめたドキュメント群です。

## 目的

現代の多くのアプリケーションは、ユーザーの利便性向上とセキュリティのためにOAuth認証（シングルサインオン）を導入しています。本ドキュメントは、異なるプロバイダ間でのOAuth認証フローの比較、必要なエンドポイント、必須パラメータ、およびユーザー情報取得方法を効率的に理解し、実装の参考にすることを目的としています。

## ドキュメント内容

各プロバイダごとの詳細なOAuth認証フローとユーザー情報取得に関するドキュメントは、`oauth/`ディレクトリ内にMarkdown形式で格納されています。各ファイルには以下の情報が含まれています。

* **OAuth認証フローの概要**: 認可エンドポイント、トークンエンドポイント、およびそれらの主要なクエリ/リクエストパラメータ。
* **Mermaid図によるフローの可視化**: 認証のステップを視覚的に理解するためのシーケンス図。
* **レスポンス例**: トークン交換やユーザー情報取得時の正常なレスポンス構造。
* **ユーザー情報取得APIエンドポイントと仕様概要**: ユーザー情報取得に使用するAPI、HTTPメソッド、必要なヘッダー、クエリパラメータ、および取得可能な情報フィールド。
* **アクセストークンとリフレッシュトークンの詳細**: 有効期限や使用上の注意点。
* **エラーハンドリング**: 発生しうる一般的なエラーとその対応方法。
